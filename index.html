Types of Content

HTML elements fall into zero or more content types. These types are:

  • Metadata content
  • Flow content
  • Sectioning content
  • Heading content
  • Phrasing content
  • Embedded content
  • Interactive content

Headings, Headers, and Footers

The h1, h2, h3, h4, h5, and h6 elements represent headings for their sections. The semantics and meaning of these elements are defined in the section on headings and outlines. These elements have a heading level given by the number in their name. The heading level corresponds to the levels of nested sections. The h1 element is for a top-level section, h2 for a subsection, h3 for a sub-subsection, and so on.

The hgroup element represents a heading and related content. The element may be used to group an h1–h6 element with one or more p elements containing content representing a subheading, alternative title, or tagline.

The header element represents a group of introductory or navigational aids. A header element is intended to usually contain a heading (an h1–h6 element or an hgroup element), but this is not required. The header header can also be used to wrap a section’s table of contents, a search form, or any relevant logos.

The footer element represents a footer for its nearest ancestor sectioning content element, or for the body element if there is no such ancestor. A footer typically contains information about its section such as who wrote it, links to related documents, copyright data, and the like.

When the footer element contains entire sections, they represent appendices, indices, long colophons, verbose license agreements, and other such content.

Contact information for the author or editor of a section belongs in an address element, possibly itself inside a footer. Bylines and other information that could be suitable for both a header or a footer can be placed in either (or neither). The primary purpose of these elements is merely to help the author write self-explanatory markup that is easy to maintain and style; they are not intended to impose specific structures on authors.

Footers don’t necessarily have to appear at the end of a section, though they usually do. When there is no ancestor sectioning content element, then it applies to the whole page.

The footer element is not itself sectioning content; it doesn’t introduce a new section.

The address element represents the contact information for its nearest article or body element ancestor. If that is the body element, then the contact information applies to the document as a whole.

The address element must not be used to represent arbitrary addresses (e.g. postal addresses), unless those addresses are in fact the relevant contact information. (The p element is the appropriate element for marking up postal addresses in general.)

The address element must not contain information other than contact information.

Typically, the address element would be included along with other information in a footer element.

Text Importance, Emphasis, and Meaning

When text needs to stress emphasis on a word or words, use the em element in HTML. This is not simply a generic italics-styled word or phrase; it is to add stress. If text is intended to stand out from the rest of the paragraph, as if it were a different mood or voice, the i element is more appropriate. Also, the em element is not intended to convey importance; for that, the strong element is more appropriate.

When text represents importance, seriousness, or urgency, the strong element should be used. The strong element can be use anywhere such as a heading, caption, or paragraph to distinguish text that is important. You can nest the strong element with more inner text being even more important.

For text that represents side commends such as small print, use the small element. This usually includes disclaimers, caveats, copyrights, term and conditions, etc. It can also be used for attribution as in satisfying a licence requirement. The small element does not de-emphasize or lower the importance of text and it is only intended for short runs of text. For instance, in a terms and conditions page, the text is the main content so small wouldn’t be appropriate. The small element should not be used for subheadings; the hgroup element is more appropriate for that.

For content that is no longer accurate or relavant, use the s element. The s element is not appropriate when indicating document edits; to mark a section of text as having been removed from a document, use the del element.

When text represents an alternate voice or mood, use the i element. This can be something like a taxonomic designation, technical term, idiomatic phrase from another language, transliteration, thought, or a ship name in Western texts. If the text needs to be stressed, use the em element. If the text is very important, use the strong element.

For text that requires attention drawn for utilitarian purposes without conveying any extra importance and with no implication of an alternate voice or mood, use the b element. It can be used for key words in a document abstract product names in a review, actionable words in interactive text-driven software, or an article lede. The b element should be used as a last resort in case no other element meets the requirements.

For text that should be highlighted for reference purposes, due to its relevance in another context, use the mark element. When used in a quotation or other block of text referred to from the prose, it indicates a highlight that was not originally present but which has been added to bring the reader’s attention to a part of the text that might not have been considered important by the original author when the block was originally written, but which is now under previously unexpected scrutiny. When used in the main prose of a document, it indicates a part of the document that has been highlighted due to its likely relevance to the user’s current activity.

For text with an unarticulated, though explicitly rendered, non-textual annotation, use the u element. This could be for labeling the text as being a proper name in Chinese text (a Chinese proper name mark), or labeling the text as being misspelt. In most cases a different element should be used.

Lists

If you want an unordered (bulleted) list, use the ul element. If you want an ordered (numbered) list, use the ol element. Child elements for either list type are indicated with the li element.

There are two ways that lists can be formatted. First, is using the built-in formatting for the ul and ol elements. This provides very functional lists but doesn’t allow for much formatting. If you need to customize the formats, you can use the CSS ::before pseudo element.

The menu element represents a toolbar consisting of its contents, in the form of an unordered list of items (represented by li elements), each of which represents a command that the user can perform or activate.

The menu element is simply a semantic alternative to ul to express an unordered list of commands (a "toolbar").

Definitions and Description Lists

When you have a term that is defined on a page, use the dfn element. The paragraph, description list group, or section that is the nearest ancestor of the dfn element must also contain the definition(s) for the term given by the dfn element.

If you have a list of name-value groups (a description list), use the dl, dt, and dd elements. The dl element defines the list, the dt element contains the name or term, and the dd element contains the description, definition, or value of the name or term. The dt element itself, when used in a dl element, does not indicate that its contents are a term being defined, but this can be indicated using the dfn element.

Abbreviations, Acronyms, and Initialisms

When the text indicates an abbreviation, acronym, or an initialism, use the abbr element. It can be used with the title attribute to provide an expansion of the abbreviation.

To understand the differences between abbreviations, acronyms, and initialisms, see the definitions below.

Abbreviation
An abbreviation is a shortened form of a word or phrase intended to speed up communication and/or use fewer symbols. An abbreviation by itself is a shortened form of a single word such as vs. (versus) or dept.(department). In addition to single-word abbreviations, there are two other types of abbreviations for compound terms: acronym and initialism.
Acronym
An acronym is a type of abbreviation made from the first letters or parts of a compound term and is read or spoken as a single word, not letter by letter. Examples are awol (absent without official leave), radar (radio detection and ranging), or scuba (self-contained underwater breating apparatus).
Initialism
An initialism is also a type of abbreviation made from the first letters or parts of a compound term, but it’s sounded letter by letter, not as one word. An example is r.p.m. (revolutions per minute).

With uncommon abbreviations, it is best practice to spell out the full word or phrase and include the abbreviation in parenthesis the first time it is used. For example, CARPE may need an introduction as Committee on Academic Rights, Privileges, and Ethics but PTA may not need an introduction as most people understand it is Parent Teacher Association.

Don’t use abbreviations that have already been taken if possible. For example, in 1959, PMS stood for primary message systems but stands premenstrual syndrome today.

When choosing an indefinite article before an abbreviation, the choice between a and an depends on how the first syllable is sounded; a vowel sound takes an an and a consonant sound takes an a. So we get an SOS, a DVD player, and a UFO.

Periods after letters. There is no consistency whether to place a period after each letter in an acronym or initialism. The current trend is to omit periods. Acronyms are spoken as a single word so it may not make sense to include the periods (UNESCO). If an initialism is made up of lowercase letters, periods are often preferred (rpm or r.p.m.; am or a.m.). However, initialisms using capital letters (ABC, ATM, HIV, IRA, SRV, URL, etc.) tend not to have periods.

Capitalization. There is no consistency on capitalization. In American English, there is a tendency to print initialisms in all-caps (FMLA, NJDEP) and acronyms in small-caps (GAAP, MADD, NASA); but this is not a rule. However, some publications use all-caps for both. In British English, the tendency is to uppercase only the first letter (Ifor = Implementation Force; Isa = individual savings account).

Pluralization. The pluralization of an initialism or acronym is done by simply adding an s without an apostrophe (CEOs, FAQs, IPOs, PCs, PINs, POWs, etc.). This is consistent with the modern trend of simplicity. Another reason to do this is that sometimes an acronym or initialism is used as a possessive, in which case you use the apostrophe (the CEO’s schedule; NASA’s budget). There are some acronyms or abbreviations that pluralize the first word if spelled out; add an s to the end of the acronym or initialism for these (POW = prisoners of war; WMDs = weapons of mass destruction). If using all lowercase acronyms or initializations, you may need to use an apostrophe for pluralization to avoid confusion (gif’s or gifs). For abbreviations that are not acronyms or initialisms, add an s before the period (paras. = paragraphs; assocs. = associates). However, there are some standard abbreviations made by doubling a prominant letter (exx. = examples; MSS. = manuscripts; pp. = pages).

Quotes and Blockquotes

When you have text that is some phrasing content from another source, you can use the q element. Quotation marks should not be used directly in the quote because the q element should automatically add the appropriate quotation marks. Sometimes you have quotation marks within quotation marks. Again, the q element (along with some CSS) will automatically take care of this.

Make some notes about quotation marks. Use Garner’s Modern English Usage. Discuss AmE vs BrE.

If the quoted content has a URL address, it can be added to the cite attribute within the q element.

The q element must not be used in place of quotation marks that do not represent quotes; for example, it is inappropriate to use the q element for marking up sarcastic statements.

As an example using the q element, according to Matthew Butterick, typography is the visual component of the written word.

If you have a section of text that is a quote from another source, you can use the blockquote element.

As with the q element, if the quoted content has a URL address, it can be added to the cite attribute within the blockquote element.

Attribution for the quotation, if any, must be placed outside the blockquote element.

You may want to consider using the blockquote element inside of a figure element.

Citations

For the title of a work (book, paper, essay, poem, song, script, film, TV show, etc.), such as the book Dracula use the cite element. This can be a work that is being quoted or referenced in detail (i.e., a citation), or it can just be a work that is mentioned in passing.

A person’s name is not the title of a work — even if people call that person a piece of work — and the element must therefore not be used to mark up people’s names. (In some cases, the b element might be appropriate for names; e.g. in a gossip article where the names of famous people are keywords rendered with a different style to draw attention to them. In other cases, if an element is really needed, the span element can be used.)

A citation is not a quote (for which the q element is appropriate).

To represent a block of computer code, the pre element can be used with a code element; to represent a block of computer output the pre element can be used with a samp element. Similarly, the kbd element can be used within a pre element to indicate text that the user is to enter.

Code

If you have a fragment of computer code, use the code element. This could be an XML element name, a filename, a computer program, or any other string that a computer would recognize.

There is no formal way to indicate the language of computer code being marked up. Computer Keystrokes

In order to represent user input, use the kbd element. This is usually for keyboard input, although it may also be used to represent other input, such as voice commands.

When the kbd element is nested inside a samp element, it represents the input as it was echoed by the system.

When the kbd element contains a samp element, it represents input based on system output, for example invoking a menu item.

When the kbd element is nested inside another kbd element, it represents an actual key or other single unit of input as appropriate for the input mechanism.

Mathematical or Coding Variables

In order to represent a mathematical or coding variable, you can use the var element. This could be an actual variable in a mathematical expression or programming context, an identifier representing a constant, a symbol identifying a physical quantity, a function parameter, or just be a term used as a placeholder in prose.

For mathematics, in particular for anything beyond the simplest of expressions, MathML is more appropriate. However, the var element can still be used to refer to specific variables that are then mentioned in MathML expressions.

Data

If you have data, you can represent it using the data element. The value attribute must be used to contain the machine-readable value of the data.

The data element can be used for several purposes.

When combined with microformats or the microdata attributes defined in this specification, the element serves to provide both a machine-readable value for the purposes of data processors, and a human-readable value for the purposes of rendering in a web browser. In this case, the format to be used in the value attribute is determined by the microformats or microdata vocabulary in use.

The element can also, however, be used in conjunction with scripts in the page, for when a script has a literal value to store alongside a human-readable value. In such cases, the format to be used depends only on the needs of the script. (The data-* attributes can also be useful in such situations.)

Subscripts/Inferiors and Superscripts/Superiors

To represent a subscript use the sub element. To represent a superscript, use the sup element.

These elements must be used only to mark up typographical conventions with specific meanings, not for typographical presentation for presentation’s sake. For example, it would be inappropriate for the sub and sup elements to be used in the name of the LaTeX document preparation system. In general, authors should use these elements only if the absence of those elements would change the meaning of the content.

The sub element can be used inside a var element, for variables that have subscripts.

Mathematical expressions often use subscripts and superscripts. Authors are encouraged to use MathML for marking up mathematics, but authors may opt to use sub and sup if detailed mathematical markup is not desired.

On thing to note is that the sub and sup elements simply shrink the size of the default font and either lower or raise its position. It is best to use a font that includes built-in subscripts and superscripts using the Opentype subs and sups features.

In addition to the Opentype subs and sups features, Opentype also specifies a sinf feature for scientific inferiors. Here are the definitions of each feature according to the official Opentype standard.

subs
The subs feature is used to present subscript forms.
An older definition is: The subs feature may replace a default glyph with a subscript glyph, or it may combine a glyph substitution with positioning adjustments for proper placement.
sups
Replaces lining or oldstyle figures with superior figures (primarily for footnote indication), and replaces lowercase letters with superior letters (primarily for abbreviated French titles).
sinf
Replaces lining or oldstyle figures (digits) with inferior figures (smaller glyphs which sit lower than the standard baseline, primarily for chemical or mathematical notation). May also replace glyphs for lowercase characters with alphabetic inferiors.

It’s challenging to determine the intended difference between the subs and sinf features. However, on the Glyphs app website, the provide a bit more direction. One difference is that subs is a change of semantic value and sinf does not allow a semantic change. In many fonts, the subs and sinf figures are identical; however, some will lower the position of sinf figures to accommodate chemical and mathematical writing a bit better.

A few examples are:

  • sub element using the Opentype subs feature: C6H12O6
  • sub element using the Opentype sinf feature: C6H12O6
  • sup element using the Opentype sups feature: x6; or BIG50
